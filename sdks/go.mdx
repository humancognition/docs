The HCE Go binding wraps the canonical `hce-core` Rust crate through the `hce-ffi` C ABI via cgo. You build the native Rust library once, import the Go package, and call `Codec` methods directly — every byte of encoding and decoding is delegated to the compiled Rust core, so output strings are byte-for-byte identical to those produced by every other HCE binding.

## Why HCE

HCE turns a 128-bit identifier into a string a human can read aloud over the phone, recognise in a log line, and type back into a form without errors — while preserving the properties you already expect from a UUID:

- **Lossless.** Every byte you put in comes out unchanged. The mapping from the input domain to the output string is bijective.
- **Format-preserving.** The output is exactly the same length for every input of the same bit width, so HCE strings fit in fixed-width columns.
- **Encrypted.** In `Sealed` and `Open` modes, a Feistel format-preserving cipher keyed by your secret randomises the bit pattern, so adjacent UUIDs do not produce adjacent HCE strings and the generation time is not leaked.
- **Integrity-checked.** Every string ends with one or more check syllables derived from HMAC-SHA256 (or CRC in `Plain` mode). A single character flip returns a non-nil error on decode before any bytes are returned.
- **Self-correcting.** A `Recover` call walks the phoneme confusion graph (`m`↔`n`, `b`↔`v`, `l`↔`r`, …) to fix a single mispronounced or mistyped syllable.
- **Chunked for working memory.** The string is split into 4–5 chunks of roughly equal length so it lands inside Cowan's 4±1 working-memory limit.

## Prerequisites

Build the `hce-ffi` shared library from the root of the HCE repository before importing the Go package. This produces `target/release/libhce_ffi.so` (Linux) or the platform equivalent.

```bash
cargo build -p hce-ffi --release
```

## Installation

Set `CGO_LDFLAGS` to point at the release directory and run `go get` as usual.

```bash
CGO_LDFLAGS="-L$(pwd)/target/release -lhce_ffi" go get github.com/humancognition/hce
```

## Constructor

```go
c, err := hce.New(key []byte, level hce.Level, mode hce.Mode)
defer c.Close()
```

<ParamField path="key" type="[]byte">
  Encryption key bytes; 32 bytes recommended. Pass a 32-byte slice for `hce.Sealed` or `hce.Open` mode. Pass `nil` for `hce.Plain`. The key is used for both the Feistel cipher and the HMAC-SHA256 check syllables.
</ParamField>

<ParamField path="level" type="hce.Level">
  Phoneme level constant: `hce.Universal`, `hce.Eu`, `hce.En`, or `hce.Numeric`. See [language levels](/concepts/language-levels) for the phoneme inventory of each level.
</ParamField>

<ParamField path="mode" type="hce.Mode">
  Codec mode constant. See the mode table below.
</ParamField>

### Modes

| Mode | Cipher | Integrity | Key required | Use case |
| --- | --- | --- | --- | --- |
| `hce.Sealed` | Feistel FPE | HMAC-SHA256 | Yes | Public URLs, API responses, anything user-facing |
| `hce.Open` | Feistel FPE | HMAC-SHA256 | Yes | Internal logs and debugging with a sortable timestamp prefix |
| `hce.Plain` | Fixed shuffle (no key) | CRC | No | base64-style use without secrets; no confidentiality |

`hce.Sealed` and `hce.Open` produce different output for the same input because the mode is mixed into the cipher tweak. `hce.Open` additionally prepends a short audible syllable derived from the embedded UUIDv7 timestamp.

### Language levels

| Level | Phoneme inventory | Best for |
| --- | --- | --- |
| `hce.Universal` | Phonemes present in ≥85% of PHOIBLE's 3,000\+ languages | Global audiences |
| `hce.Eu` | Universal plus EU-language consonant clusters | European product surfaces |
| `hce.En` | Adds English-specific clusters (e.g. `st`, `sp`, `sk`) | English-only audiences |
| `hce.Numeric` | Digit substitution for the consonant pool | Voice IVR, numeric keypads |

## Core Methods

| Method | Returns | Description |
| --- | --- | --- |
| `Encode(data []byte)` | `(string, error)` | Encode bytes to an HCE string |
| `Decode(input string)` | `([]byte, error)` | Decode to original bytes |
| `Recover(input string)` | `([]byte, error)` | Attempt phoneme confusion correction on a corrupted string |

`Decode` is strict by design: a single character flip returns a non-nil error rather than silently returning the wrong bytes. If you expect human input — handwriting, telephony, voice transcription — call `Recover` first to attempt automatic correction across the phoneme confusion graph.

## Configuration Methods

Call these methods on your `*Codec` before calling `Encode` or `Decode`.

| Method | Description |
| --- | --- |
| `WithBitWidth(bits uint32)` | Domain bit width (16–128) |
| `WithCase(cas hce.Case)` | Output case |
| `WithCheckSyllables(n int)` | HMAC check syllables (1–8) |
| `WithSeparator(sep byte)` | Separator character between chunks |
| `WithChunkNone()` | No separator — contiguous output string |
| `WithChunkFixed(charSize int)` | Fixed syllable count per chunk |
| `WithChunkPattern(pattern []int)` | Custom syllable-per-chunk pattern |
| `WithTimestampConfig(epochMs int64, gran hce.Granularity)` | Timestamp prefix (`hce.Open` mode only) |

## Full Example

```go
package main

import (
    "fmt"
    hce "github.com/humancognition/hce"
)

func main() {
    key := []byte("32-byte-secret-key-here-xxxxxx!!")
    uuid := []byte{1, 149, 227, 160, 124, 46, 123, 65, 143, 61, 154, 108, 30, 11, 77, 39}

    c, err := hce.New(key, hce.Universal, hce.Sealed)
    if err != nil {
        panic(err)
    }
    defer c.Close()

    encoded, err := c.Encode(uuid)
    if err != nil {
        panic(err)
    }
    fmt.Println(encoded)

    decoded, err := c.Decode(encoded)
    if err != nil {
        panic(err)
    }
    fmt.Printf("%x\n", decoded)
}
```

## Self-Recovery Example

If a user mistypes or mispronounces a single syllable (`l`↔`r`, `m`↔`n`, `b`↔`v`), `Recover` walks the phoneme confusion graph and returns the corrected bytes when there is exactly one valid candidate. If several candidates are valid, or none are, it returns an error.

```go
c, _ := hce.New(key, hce.Universal, hce.Sealed)
defer c.Close()

// User typed "KLUMA-..." instead of "KRUMA-..."
decoded, err := c.Recover("KLUMA-...")
if err != nil {
    // Ambiguous or unrecoverable input — ask the user to re-enter
}
```

## Running Tests

Set `LD_LIBRARY_PATH` so the Go test runner can locate the shared library, then run the standard test command.

```bash
LD_LIBRARY_PATH=target/release go test -v ./...
```

<Warning>
  The `Codec` struct holds a native Rust allocation. Always call `c.Close()` when you are done — or use `defer c.Close()` immediately after a successful `hce.New` call — to release the underlying memory and avoid leaks.
</Warning>
