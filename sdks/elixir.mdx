The HCE Elixir binding is a Rustler NIF (Native Implemented Function) that calls directly into the canonical `hce-core` Rust crate. You add it as a Mix dependency, compile the native extension once, and use the `Hce` module like any other Elixir module. The API follows Elixir conventions — `new/3` returns `{:ok, codec}`, `decode/2` returns `{:ok, binary} | {:error, term}`, and configuration functions accept and return the codec struct for easy piping. Output strings are byte-for-byte identical to those produced by every other HCE binding.

## Why HCE

HCE turns a 128-bit identifier into a string a human can read aloud over the phone, recognise in a log line, and type back into a form without errors — while preserving the properties you already expect from a UUID:

- **Lossless.** Every byte you put in comes out unchanged. The mapping from the input domain to the output string is bijective.
- **Format-preserving.** The output is exactly the same length for every input of the same bit width, so HCE strings fit in fixed-width Ecto columns.
- **Encrypted.** In `:sealed` and `:open` modes, a Feistel format-preserving cipher keyed by your secret randomises the bit pattern, so adjacent UUIDs do not produce adjacent HCE strings and the generation time is not leaked.
- **Integrity-checked.** Every string ends with one or more check syllables derived from HMAC-SHA256 (or CRC in `:plain` mode). A single character flip returns `{:error, reason}` on decode before any bytes are returned.
- **Self-correcting.** `recover/2` walks the phoneme confusion graph (`m`↔`n`, `b`↔`v`, `l`↔`r`, …) to fix a single mispronounced or mistyped syllable.
- **Chunked for working memory.** The string is split into 4–5 chunks of roughly equal length so it lands inside Cowan's 4±1 working-memory limit.

## Prerequisites

Build the `hce-ffi` shared library before running `mix deps.compile`. The Rustler NIF links against it at compile time.

```bash
cargo build -p hce-ffi --release
```

## Installation

Add the dependency to your `mix.exs` file and run `mix deps.get`.

```elixir
def deps do
  [{:hce, path: "bindings/elixir"}]
end
```

## API

### Creating a codec

```elixir
Hce.new(key, level \\ :universal, mode \\ :sealed)
```

`key` is a binary string (32 bytes recommended). `level` is one of `:universal`, `:eu`, `:en`, or `:numeric`. `mode` is one of `:sealed`, `:open`, or `:plain`. Returns `{:ok, %Hce{}}` on success or `{:error, reason}` if the key is missing for an encrypted mode.

### Modes

| Mode | Cipher | Integrity | Key required | Use case |
| --- | --- | --- | --- | --- |
| `:sealed` | Feistel FPE | HMAC-SHA256 | Yes | Public URLs, API responses, anything user-facing |
| `:open` | Feistel FPE | HMAC-SHA256 | Yes | Internal logs and debugging with a sortable timestamp prefix |
| `:plain` | Fixed shuffle (no key) | CRC | No | base64-style use without secrets; no confidentiality |

`:sealed` and `:open` produce different output for the same input because the mode is mixed into the cipher tweak. `:open` additionally prepends a short audible syllable derived from the embedded UUIDv7 timestamp.

### Language levels

| Level | Phoneme inventory | Best for |
| --- | --- | --- |
| `:universal` | Phonemes present in ≥85% of PHOIBLE's 3,000\+ languages | Global audiences |
| `:eu` | Universal plus EU-language consonant clusters | European product surfaces |
| `:en` | Adds English-specific clusters (e.g. `st`, `sp`, `sk`) | English-only audiences |
| `:numeric` | Digit substitution for the consonant pool | Voice IVR, numeric keypads |

See [language levels](/concepts/language-levels) for the full phoneme inventory of each level.

### Encoding and decoding

```elixir
Hce.encode(codec, data :: binary) :: String.t()
Hce.decode(codec, input :: String.t()) :: {:ok, binary} | {:error, term}
Hce.recover(codec, input :: String.t()) :: {:ok, binary} | {:error, term}
```

`decode/2` is strict by design: a single character flip returns `{:error, :integrity_failure}` rather than silently returning the wrong bytes. If you expect human input — handwriting, telephony, voice transcription — call `recover/2` first to attempt automatic correction across the phoneme confusion graph (`m`↔`n`, `b`↔`v`, `l`↔`r`, …). `recover/2` returns `{:ok, binary}` when exactly one candidate is valid, or `{:error, reason}` if the input is ambiguous or unrecoverable.

### Configuration functions

All configuration functions accept the codec as the first argument and return the updated codec, so you can pipe them.

```elixir
Hce.with_bit_width(codec, bits :: integer)
Hce.with_modulus(codec, modulus :: integer)
Hce.with_cipher_feistel(codec, key :: binary | nil)
Hce.with_cipher_shuffle(codec)
Hce.with_case(codec, :upper | :lower)
Hce.with_check_syllables(codec, n :: integer)
Hce.with_separator(codec, sep :: String.t())
Hce.with_chunk_none(codec)
Hce.with_chunk_fixed(codec, n :: integer)
Hce.with_chunk_pattern(codec, pattern :: [integer])
Hce.with_timestamp_config(codec, epoch_ms :: integer, :second | :minute | :hour | :day | :week | :month)
```

### Releasing resources

```elixir
Hce.destroy(codec)
```

Call `Hce.destroy/1` when you are done with a codec to release its native memory immediately.

## Example

The example below encodes a 16-byte UUID, decodes it back, and prints both results.

```elixir
{:ok, h} = Hce.new("32-byte-key-here-xxxxxx!!", :universal, :sealed)

uuid = <<1, 149, 227, 160, 124, 46, 123, 65, 143, 61, 154, 108, 30, 11, 77, 39>>
encoded = Hce.encode(h, uuid)
IO.puts(encoded)
# => PETREN-NISLORPEN-LAFLER-SRORGULGOLFUN-PREPLEN

{:ok, decoded} = Hce.decode(h, encoded)
IO.inspect(decoded, base: :hex)
# => <<1, 149, 227, 160, 124, 46, 123, 65, 143, 61, 154, 108, 30, 11, 77, 39>>

Hce.destroy(h)
```

<Warning>
  You must call `Hce.destroy/1` to release the native resource when you are finished with a codec. The BEAM garbage collector does not manage Rust-allocated memory, so codecs that are never destroyed will leak for the lifetime of the VM node.
</Warning>

<Tip>
  In a Phoenix controller, pattern-match on `Hce.decode/2` directly in the function head or with a `with` chain to return structured error responses without nested `case` blocks:

  ```elixir
  with {:ok, codec} <- Hce.new(key, :universal, :sealed),
       {:ok, binary_id} <- Hce.decode(codec, hce_param) do
    render(conn, :show, item: Repo.get!(Item, binary_id))
  else
    {:error, reason} -> conn |> put_status(:bad_request) |> json(%{error: reason})
  end
  ```
</Tip>
