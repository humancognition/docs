HCE is a lossless codec that transforms raw binary identifiers — UUIDs, ULIDs, Snowflakes, and any 16-byte value — into encrypted, pronounceable strings like `PETREN-NISLORPEN-LAFLER-SRORGULGOLFUN-PREPLEN`. The encoding is fully reversible: decrypt it back to the original bytes with the same key at any time. HCE ships as a Rust core library with official bindings for Python, JavaScript/WASM, Go, Java, Swift, Elixir, and MySQL/MariaDB.

## Why HCE?

Raw identifiers like UUIDs expose structure (version bits, timestamps) and are hard to communicate verbally. HCE solves both problems simultaneously:

<CardGroup cols={3}>
  <Card title="Encrypted" icon="lock">
    HMAC-SHA256 Feistel FPE with 8 rounds hides internal identifier structure from observers.
  </Card>
  <Card title="Pronounceable" icon="microphone">
    35 phonemic onsets × 20 vowel nuclei produce syllables that read naturally in multiple languages.
  </Card>
  <Card title="Lossless" icon="arrows-rotate">
    Every encoded string decodes back to the exact original bytes — no information is lost.
  </Card>
  <Card title="Self-Recovering" icon="wrench">
    Built-in confusion matrix corrects common mishearings like m/n or al/an automatically.
  </Card>
  <Card title="Format-Preserving" icon="ruler">
    A 128-bit input always produces a predictable ~50-character output with configurable chunking.
  </Card>
  <Card title="Multi-Platform" icon="globe">
    One core, eight languages: Rust, Python, JS/WASM, Go, Java, Swift, Elixir, and SQL.
  </Card>
</CardGroup>

## Quick Example

```rust
use hce_core::{Hce, HceMode, LanguageLevel};

let hce = Hce::new(
    Some(b"32-byte-secret-key-here-xxxxxx!!"),
    LanguageLevel::Universal,
    HceMode::Sealed,
);

// Encode a UUID's raw bytes
let encoded = hce.encode(&uuid_bytes);
// => "PETREN-NISLORPEN-LAFLER-SRORGULGOLFUN-PREPLEN"

// Decode back to the original bytes
let decoded = hce.decode(&encoded).unwrap();
assert_eq!(decoded, uuid_bytes);
```

## Get Started

<Steps>
  <Step title="Install HCE">
    Pick your language and add HCE to your project — see the [Installation guide](/installation) for all package managers.
  </Step>
  <Step title="Choose a mode">
    Use `Sealed` for confidentiality, `Open` for sortable timestamp-prefixed IDs, or `Plain` for CRC-only integrity. See [Modes](/concepts/modes).
  </Step>
  <Step title="Encode and decode">
    Call `encode()` with your raw identifier bytes. Store or transmit the string, then call `decode()` with the same key to recover the original bytes.
  </Step>
  <Step title="Explore adapters">
    Use the built-in [UUID](/adapters/uuid), [ULID, Snowflake, and Xid adapters](/adapters/ulid-snowflake-xid) to skip manual byte extraction.
  </Step>
</Steps>
