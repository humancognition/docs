title: "Get Started with HCE: Encode Your First Identifier"
sidebarTitle: "Quickstart"
description: "Install HCE, create a codec with a secret key, encode a UUID into a pronounceable string, and decode it back to the original bytes."
---

This guide walks you through everything you need to encode your first identifier and decode it back to its original bytes. By the end you will have a working codec, a real encoded output, and the confidence to wire HCE into your own project. The examples use Rust for the main walkthrough, with equivalent install commands for Python and JavaScript shown alongside.

<Steps>

  <Step title="Install HCE">
    Add HCE to your project using the package manager for your language.

    <CodeGroup>

    ```toml Rust (Cargo.toml)
    [dependencies]
    hce-core = "0.1"

    # Optional: adapters for UUID, ULID, Snowflake, Xid
    hce-adapters = "0.1"

    # Optional: additional cipher backends (Feistel, Shuffle)
    hce-fpe = "0.1"
    ```

    ```bash Python
    pip install hce
    ```

    ```bash JavaScript / Node.js
    npm install @humancognition/hce
    ```

    </CodeGroup>

    If you want the command-line tool alongside your project, install it globally with Cargo:

    ```bash
    cargo install hce-cli
    ```

  </Step>

  <Step title="Create a Codec">
    Instantiate `Hce` with a 32-byte secret key, a language level, and an operating mode. The `Sealed` mode encrypts the identifier fully and attaches an HMAC check syllable — the right choice for most production use cases.

    ```rust
    use hce_core::{Hce, HceMode, LanguageLevel};

    let hce = Hce::new(
        Some(b"32-byte-secret-key-here-xxxxxx!!"),
        LanguageLevel::Universal,
        HceMode::Sealed,
    );
    ```

    `LanguageLevel::Universal` selects the cross-lingual phoneme set built from PHOIBLE data — phoneme pairs that appear in at least 40% of surveyed languages — so the output is pronounceable across most major languages. If your audience is English-only, `LanguageLevel::En` produces slightly more English-natural syllables.

  </Step>

  <Step title="Encode an Identifier">
    Pass your raw identifier bytes to `encode`. The method runs the Feistel FPE cipher, maps the ciphertext to phonemic syllables, appends a check syllable, and returns a dash-separated string.

    ```rust
    let id = hce.encode(&[1, 149, 227, 160, 124, 46, 123, 65,
                           143, 61, 154, 108, 30, 11, 77, 39]);

    println!("{}", id);
    // => "PETREN-NISLORPEN-LAFLER-SRORGULGOLFUN-PREPLEN"
    ```

    The output is uppercase by default. You can call `.with_case(HceCase::Lower)` on the codec to get lowercase, or `.with_separator('/')` to change the chunk delimiter.

  </Step>

  <Step title="Decode Back to Bytes">
    Pass the encoded string to `decode`. HCE verifies the check syllable, strips it, reverses the phoneme-to-integer mapping, and applies the inverse Feistel cipher to recover the original bytes exactly.

    ```rust
    let bytes = hce.decode(&id).unwrap();

    assert_eq!(
        bytes,
        vec![1, 149, 227, 160, 124, 46, 123, 65,
             143, 61, 154, 108, 30, 11, 77, 39]
    );
    ```

    `decode` returns a `Result`. If the string was modified — even a single character changed — the HMAC check syllable will not match and `decode` returns an `Err`. You can call `hce.recover(&id)` instead to attempt automatic self-correction of common phoneme transcription errors before hard-failing.

  </Step>

</Steps>

<Note>
  **Key management matters.** Your 32-byte key is the secret that makes encoded identifiers opaque. Anyone who holds the key can decode any string produced with it. Store your key in an environment variable or secrets manager — never hard-code it in source. If you need to rotate keys, re-encode your existing identifiers before retiring the old key, because decoding always requires the same key that was used to encode.
</Note>
