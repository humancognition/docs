The `hce` CLI lets you encode, decode, and recover identifiers directly from the terminal — useful for scripts, CI pipelines, and quick one-off conversions. You pass hex-encoded bytes in and get back a pronounceable, encrypted HCE string that travels safely through voice channels, tickets, and logs.

## Installation

Install the CLI from [crates.io](https://crates.io/crates/hce-cli) using Cargo:

```bash
cargo install hce-cli
```

Once installed, run `hce --help` to confirm the binary is on your `PATH`.

## Basic Usage

The three commands you will reach for most often are `encode`, `decode`, and `recover`. The examples below all use a UUID (expressed as hex bytes) as the input identifier.

```bash
# Encode a UUID (hex input)
echo "0195e3a07c2e7b418f3d9a6c1e0b4d27" | hce encode --key "<hex-key>"

# Decode back
hce decode --key "<hex-key>" "PETREN-NISLORPEN-LAFLER-SRORGULGOLFUN-PREPLEN"

# Recover a possibly-misheard identifier
hce recover --key "<hex-key>" "PETREN-NISLORPEM-LAFLER-SRORGULGOLFUN-PREPLEN"
```

## Key Format

Keys must be passed as hex-encoded strings. The recommended approach is to generate a cryptographically random 32-byte key and convert it to hex in one step:

```bash
# Generate a random 32-byte key and hex-encode it
openssl rand -hex 32
```

Store the resulting 64-character string somewhere safe (for example, a secrets manager or an environment variable) and pass it via `--key` whenever you call `hce encode` or `hce decode`.

<Note>
  Input to `hce encode` is always hex-encoded bytes, not raw text. The UUID bytes `0195e3a0...` shown in the examples are already in hex. If you are starting from a raw UUID string like `0195e3a0-7c2e-7b41-8f3d-9a6c1e0b4d27`, strip the dashes first: `echo "0195e3a07c2e7b418f3d9a6c1e0b4d27" | hce encode --key "<hex-key>"`.
</Note>

<Warning>
  Keys passed via `--key` appear as plain text in your shell history and in process listings. In production environments, read the key from an environment variable or a secrets manager and pass it programmatically rather than typing it directly on the command line.
</Warning>
